## 주요 발견 사항

### 1. [높음] 장시간 세션에서 검색 기능이 전체 자막이 아니라 "현재 렌더된 500개"만 검색한다

- 당시 근거:
  - 렌더링 대상이 `Config.MAX_RENDER_ENTRIES` 기준으로 최근 500개로 제한되어 있었고, 검색은 렌더 텍스트 기준으로 수행됐다.
- 현재 상태:
  - 해결됨. 검색은 `self.subtitles` 전체 스냅샷 기준으로 수행되고, 검색 결과 이동 시 해당 entry가 렌더 범위에 포함되도록 동적으로 offset을 조정한다.
- 추가 테스트:
  - `tests/test_feature_plan_20260325.py`에서 600개 초과 자막 검색과 tail 복귀를 검증한다.

### 2. [높음] 추출 중 상태를 갈아끼우는 기능들에 실행 중 가드가 부족하다

- 당시 근거:
  - 편집/삭제만 일부 차단되고, 세션 불러오기/병합/리플로우/전체 삭제 등은 `capture_state`를 교체할 수 있었다.
- 현재 상태:
  - 해결됨. 세션 불러오기, DB 세션 로드, 병합, 줄넘김 정리, 내용 지우기, 전체 자막 삭제, 편집, 삭제를 공통 runtime mutation guard로 차단한다.
  - 관련 QAction/QPushButton도 추출 중에는 disable 된다.
- 추가 테스트:
  - `tests/test_feature_plan_20260325.py`에서 실행 중 차단과 상태 보존을 검증한다.

### 3. [중간] DB 세션 불러오기와 파일 세션 불러오기가 서로 다른 수준의 메타데이터를 복원한다

- 당시 근거:
  - DB 로드 경로는 `subtitles` 중심, 파일 로드는 `version`/`url`/버전 경고까지 복원했다.
- 현재 상태:
  - 해결됨. 파일/DB 세션 로드는 공통 payload와 `_complete_loaded_session()`을 공유하며, `version`, `url`, `committee_name`, `created_at`, `subtitles`, `skipped`를 동일하게 처리한다.

### 4. [중간] 프리셋 내보내기/가져오기가 비대칭이라 round-trip이 완전하지 않다

- 당시 근거:
  - export는 `committee`와 `custom`을 모두 저장하지만 import는 `custom`만 반영했다.
- 현재 상태:
  - 해결됨. import/export 모두 `committee` + `custom`을 round-trip하고, import 시 동일 키는 가져온 값으로 overwrite 한다.

### 5. [중간] 저장 경로 대부분은 원자적 저장인데 일부 내보내기만 예외라 파일 손상 내성이 들쭉날쭉하다

- 당시 근거:
  - 통계와 프리셋 export가 일반 `open()`/`json.dump()` 경로를 사용했다.
- 현재 상태:
  - 해결됨. 통계 export는 `atomic_write_text`, 프리셋 export는 `atomic_write_json`을 사용한다.

## 추가 구현 권장 사항

### 1. DB 검색 결과를 "보기 전용 목록"에서 "후속 행동 가능한 결과"로 확장할 필요가 있다

- 현재 상태:
  - 해결됨. 검색 결과 다이얼로그는 `세션 불러오기`와 `결과로 이동`을 제공하며, 결과 이동은 DB `sequence` 기준으로 처리된다.

### 2. 세션 병합의 중복 제거 정책은 운영 옵션으로 노출하는 편이 안전하다

- 현재 상태:
  - 해결됨. 병합 UI에서 dedupe 기준을 `보수적(같은 초)`와 `기존(30초 버킷)` 중 선택할 수 있다.

### 3. 불러온 세션에 이어서 바로 캡처를 계속하는 워크플로는 현재 없다

- 현재 상태:
  - 이번 계획 범위에서 의도적으로 제외됨. 시작 시 기존 자막/상태를 초기화하는 정책은 그대로 유지된다.
