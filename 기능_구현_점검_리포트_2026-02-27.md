## 적용 완료 상태 (v16.13, 2026-02-27)

아래 7개 항목은 코드 기준으로 반영 완료되었습니다.

1. **`xcgcd` 자동 감지 경로 연결 완료**
   - `_extraction_worker` 시작 직후 및 재연결 경로에서 `_detect_live_broadcast`를 실제 호출
   - 감지된 URL을 동일 `url` 변수로 재사용해 경로 일관성 유지

2. **keepalive 경로 활성화 + finalize dead path 최소 정리 완료**
   - 동일 `raw_compact` 유지 시 keepalive 메시지 주기 발행
   - `end_time` 연장 로직이 실제 동작하도록 연결
   - 사용 경로가 없는 finalize timer 관련 코드 최소 정리

3. **로그 경로 정책 통일 완료**
   - `core/logging_utils.py`가 상대 경로 대신 `Config.LOG_DIR` 사용

4. **저장 안정성 보강 완료**
   - 저장 스레드 추적(`_active_save_threads`) 및 종료 시 제한시간 대기 추가
   - TXT/SRT/VTT/RTF에 `atomic_write_text` 적용 (임시파일 후 교체)

5. **LiveBroadcastDialog 종료 안정성 보강 완료**
   - `done`/`closeEvent` 모두 `_shutdown_fetch_thread()`로 통합
   - 대기시간을 `Config.API_TIMEOUT + 1초`로 조정

6. **짧은 발화 정책 반영 완료 (`허용 + 노이즈 필터`)**
   - `is_meaningful_subtitle_text` 도입
   - 한글/영문 1~2자 발화 허용, 숫자/기호-only 차단

7. **테스트 보강 완료**
   - `test_short_utterance_filter.py` 신규 추가
   - 워커 안정성/파일 I/O 안전성 관련 테스트 확장
   - 회귀 포함 전체 테스트 통과 (`24 passed`)

## 초기 점검 당시 잠재 이슈 및 보완 필요 사항

### 1) [높음] 생중계 자동 감지 로직이 시작 경로에 미연결
- 근거
  - 자동 감지 메서드는 구현되어 있음: `ui/main_window.py:2055` (`_detect_live_broadcast`)
  - 실제 워커 시작 경로는 `driver.get(url)`로 직접 진입: `ui/main_window.py:2531`
  - `_detect_live_broadcast` 호출 지점이 없음 (정의 외 참조 부재)
- 문서 대비
  - README에서 시작 상태로 `현재 생중계 감지 중`, `생중계 감지 성공`을 안내: `README.md:154`, `README.md:155`
  - CLAUDE 문서에서 `xcode`만으로 자동 보완/감지 지원 명시: `CLAUDE.md:183`
- 영향
  - `xcode`만 입력된 URL에서 실제 진행 방송(`xcgcd`) 미보완 가능
  - 방송 전환/리다이렉트 상황에서 자막 미수집 확률 증가
- 권장 조치
  - 워커 초기화 직후 `_detect_live_broadcast`를 호출하여 최종 URL 확정 후 `driver.get(final_url)` 수행
  - 해당 흐름에 대한 통합 테스트 추가

### 2) [높음] keepalive/finalize 파이프라인이 사실상 비활성
- 근거
  - keepalive 처리기 존재: `ui/main_window.py:3826`
  - 메시지 처리 분기 존재: `ui/main_window.py:3636`
  - 그러나 keepalive enqueue 코드 없음 (`message_queue.put(("keepalive", ...))` 부재)
  - `last_subtitle`는 초기화/정리만 있고 실질 할당 경로가 없음: `ui/main_window.py:122`, `ui/main_window.py:1722`, `ui/main_window.py:3517`
  - `finalize_timer`는 생성되지만 start 호출 없음: `ui/main_window.py:242`
- 문서 대비
  - CLAUDE 문서의 `동일 자막 유지 시 end_time 주기 갱신` 설명과 불일치: `CLAUDE.md:87`
- 영향
  - 자막이 오래 유지되는 구간의 `end_time` 갱신 누락 가능
  - SRT/VTT 구간 길이 정확도 저하 가능
- 권장 조치
  - 워커에서 동일 raw 유지 시 keepalive 메시지 전송 재도입 또는
  - 해당 dead 경로 제거 후, 현재 아키텍처 기준으로 end_time 갱신 정책 단일화

### 3) [중간] 로그 경로가 앱 기준 경로 정책과 불일치
- 근거
  - 앱 기준 로그 경로 상수 존재: `core/config.py:62` (`LOG_DIR`)
  - 실제 로깅은 상대경로 `"logs"` 사용: `core/logging_utils.py:13`
- 영향
  - 실행 위치(CWD)에 따라 로그가 흩어짐
  - 문서의 "앱 기준 경로 저장" 정책과 불일치
- 권장 조치
  - `core/logging_utils.py`에서 `Config.LOG_DIR` 사용하도록 통일

### 4) [중간] 백그라운드 저장 스레드 daemon 종료로 파일 손상 위험
- 근거
  - 저장 스레드 daemon 실행: `ui/main_window.py:4821`
  - 종료 시 저장 스레드 join/완료 대기 없음: `ui/main_window.py:6447`
- 영향
  - 앱 종료 타이밍에 저장 중이면 파일이 부분 저장될 수 있음
- 권장 조치
  - active save thread/future 추적 후 `closeEvent`에서 제한시간 대기
  - 텍스트 기반 저장(TXT/SRT/VTT/RTF)에도 원자적 저장(임시파일 후 replace) 적용 고려

### 5) [중간] LiveBroadcastDialog 스레드 종료 타임아웃이 네트워크 타임아웃보다 짧음
- 근거
  - API 호출 timeout: `ui/dialogs.py:27` (`Config.API_TIMEOUT`, 기본 5초)
  - 다이얼로그 종료 시 QThread 대기: 1초만 wait: `ui/dialogs.py:194`, `ui/dialogs.py:214`
- 영향
  - 빠른 닫기 시 워커가 살아있는 상태로 다이얼로그 파기될 위험
  - 환경에 따라 `QThread` 관련 경고/불안정 동작 가능
- 권장 조치
  - 인터럽트 플래그 + 종료 대기시간을 API timeout 이상으로 조정
  - 또는 Qt 네트워크 비동기 API로 전환

  - 짧은 발화 회귀 테스트 추가
