## ✨ v16.11 새 기능

### 🧠 자막 수집 알고리즘 완벽 재설계 (v16.11)
- **글로벌 히스토리 Suffix 매칭** - 확정된 모든 텍스트를 누적하고, 새 raw에서 히스토리의 마지막 부분(suffix) 이후만 추출
- **DOM 루핑 면역** - 웹사이트 특유의 텍스트 반복/루핑 현상 완벽 대응
- **자막 반복 완전 해결** - 기존 앵커 기반, 버퍼 비교, 문장 해시 방식 모두 실패 → 새 알고리즘으로 해결

### 🛡️ 파이프라인 안정화 고정 (v16.11 유지보수)
- **코어 알고리즘 고정** - `_process_raw_text` / `_extract_new_part`의 글로벌 히스토리 + suffix 의미론은 변경하지 않음
- **입력 게이트 레이어** - `preview`는 `_prepare_preview_raw`를 통과한 뒤 코어 알고리즘으로 전달
- **다단계 fallback** - suffix desync/ambiguous 상황에서 `_extract_stream_delta`, `_slice_incremental_part`로 증분 우선 추출
- **안전한 재동기화** - 반복 실패 시에만 제한적으로 `_confirmed_compact`/`_trailing_suffix` 리셋
- **누락 완화** - 종료 직전 큐 소진 시 강제 플러시 경로로 마지막 자막 누락 최소화

### 📝 타임스탬프 저장 개선 (v16.11)
- **1분 간격 표시** - TXT/DOCX/HWP 저장 시 타임스탬프가 1분 이상 변경될 때만 표시 (메인 윈도우와 동일)
- **가독성 향상** - 매 줄마다 타임스탬프가 나오지 않아 읽기 편함

### 🐛 버그 수정 (v16.11)
- **줄넘김 정리 후 자막 수집 중단 수정** - Reflow 후 글로벌 히스토리 리셋 추가
- **end_time null 오류 수정** - Reflow된 자막의 end_time이 None일 때 발생하는 TypeError 방어

## ✨ v16.10 기능

### ⚡ 스트리밍 자막 엔진 탑재 (v16.10)
- **실시간 반응성** - 2초 대기 없이 화면에 자막이 나오는 즉시 프로그램에 반영
- **데이터 유실 방지** - 자막이 1분 이상 누적되어도 사라지지 않고(Disappearing issue 해결) 안정적으로 수집
- **단어 증분 누적 (Accumulation)** - 문장이 잘리거나 겹칠 때 단어 단위로 자연스럽게 이어붙여 불필요한 줄바꿈 방지
- **타임라인 자동 분할** - 문장이 일정 길이(150자)를 초과하면 자동으로 잘라 새 타임스탬프를 생성, 가독성 확보
- **중지 안전성 강화** - '중지' 버튼 클릭 시 프로그램이 강제 종료되지 않도록 예외 처리 강화

### 🎨 UI 개선 (v16.10)
- **상단 접기/펼치기** - 자막 영역을 더 넓게 쓸 수 있도록 상단 메뉴 접기 기능 개선
- **버튼 UI 최적화** - 글자가 잘리던 버튼들의 너비 조정 (생중계 목록, 태그 등)
- **키워드 설정 편의성** - 설정 패널에서 키워드를 바로 입력하고 캐싱하도록 개선

### 🛡️ 저장/종료 안정성 개선 (v16.10)
- **중지 시 누락 방지** - 종료 직전 `preview` 큐를 소진하고 마지막 버퍼를 확정
- **DOCX/HWP 비동기 저장** - 대용량 저장 시에도 UI 멈춤 없이 백그라운드 처리
- **HWP 저장 재시도/검증** - 저장 실패 시 재시도 후 대체 형식 안내
- **손상 항목 자동 건너뜀** - 세션/병합 로드 중 오류 항목은 로그에 남기고 계속 진행
- **실시간 저장 오류 감지** - 디스크 오류 등으로 저장 실패 시 3회 재시도 후 경고 알림

### 🚀 성능 최적화 (v16.10)
- **대량 자막 렌더링 최적화** - 자막이 500개를 넘어가면 화면에는 최근 자막만 표시하여 렉 방지 (저장은 전체 됨)
- **텍스트 처리 효율화** - 긴 문장의 비교/분석 알고리즘 개선으로 CPU 사용량 감소
- **DB 검색 호환성** - 특수문자 검색 시 오류 없이 일반 검색으로 자동 전환

## ✨ v16.9 이전 기능

### 🛡️ 데이터 안정성 (v16.9)
- **자동 백업 백그라운드 처리** - UI 프리즈 없이 파일 I/O 수행
- **중복 백업 방지** - 백업 중복 실행 방지 락 적용
- **세션 로드 복구** - JSON 파싱 오류 시 백업 폴더 열기 안내

### 🔒 안정성 및 성능 (v16.9)
- **스레드 안전성 전면 강화** - 자막 저장/편집/초기화 시 `subtitle_lock` 적용으로 충돌 방지
- **DB 최적화** - 연결 캐싱(Pooling) 및 Bulk Insert 최적화, FTS5 검색 엔진 적용
- **자막 렌더링 가속** - 전체 다시 그리기 대신 증분 렌더링 도입
- **스크롤 우선 표시** - 사용자가 스크롤로 보고 있을 때 자동 스크롤을 일시 중지하고 위치 유지
- **길이/간격 기반 병합** - 지나치게 길거나 간격이 긴 자막은 분리 저장
- **중지 즉시 확정** - 중지 시 진행 중인 자막을 즉시 확정해 종료 안정성 개선

### 🎨 UI/UX 개선 (v16.9)
- **키워드 입력 최적화** - 디바운싱(Debouncing) 적용으로 입력 중 렉 제거
- **병합 옵션 세분화** - 병합 시 현재 세션 자막 보존 여부 선택 가능
- **HWP 저장 개선** - 저장 실패 시 RTF/DOCX 등 대체 포맷 제안

### 📡 생중계 목록 선택 (v16.7)

- **선택 즉시 접속** - 선택한 방송의 `xcode+xcgcd` URL 자동 구성

## 🚀 사용 방법

### 기본 사용법

#### 1단계: 프로그램 실행
```bash
python "국회의사중계 자막.py"
```

#### 2단계: 위원회 선택
세 가지 방법 중 선택:

**방법 A: 프리셋 사용 (권장)**
1. `📋 상임위` 버튼 클릭
2. 원하는 위원회 선택 (예: 본회의, 법제사법위원회)
3. URL이 자동으로 입력됨

**방법 B: 직접 URL 입력**
1. URL 입력창에 국회 의사중계 URL 입력
2. 예: `https://assembly.webcast.go.kr/main/player.asp?xcode=10`

**방법 C: 생중계 목록 선택 (v16.7)**
1. `📡 생중계 목록` 버튼 클릭
2. 목록에서 현재 방송 선택
3. URL이 자동으로 입력됨

#### 3단계: 옵션 설정
- ✅ **자동 스크롤**: 새 자막이 추가될 때 자동으로 아래로 스크롤 (사용자 스크롤 시 일시 중지)
- ✅ **실시간 저장**: 자막을 파일에 실시간으로 저장 (realtime_output 폴더)
- ✅ **헤드리스 모드**: 브라우저 창을 숨기고 백그라운드에서 실행

#### 4단계: 추출 시작
- `▶ 시작` 버튼 클릭 또는 **F5** 키
- 상태바에서 진행 상황 확인:
  - `🔍 현재 생중계 감지 중...` - xcgcd 자동 탐지 진행
  - `✅ 생중계 감지 성공!` - 오늘 생중계 발견
  - `자막 모니터링 중` - 정상 동작 중
- 연결 상태 표시: 🟢 연결됨, 🔴 끊김, 🟡 재연결 중

#### 5단계: 자막 확인
- 왼쪽 패널에 자막이 실시간으로 표시됨
- 오른쪽 통계 패널에서 글자수, 단어수, 실행 시간 확인
- `⏳` 표시는 아직 확정되지 않은 진행 중 자막

#### 6단계: 저장
추출 완료 후 다양한 형식으로 저장 (자동 파일명 제안됨):
- **파일 → TXT 저장** (Ctrl+S) - 일반 텍스트
- **파일 → SRT 저장** - 자막 파일 형식
- **파일 → DOCX 저장** - Word 문서
- **파일 → 세션 저장** (Ctrl+Shift+S) - 나중에 다시 불러오기 가능 + DB 저장

### 고급 사용법

#### 데이터베이스 활용 (v16.6 신규)
1. **세션 히스토리**: `데이터베이스` → `📋 세션 히스토리`

#### 자막 병합 (v16.6 신규)
1. **도구 → 📎 자막 병합...** (Ctrl+Shift+M)
2. 병합할 세션 파일들 추가
3. 옵션 선택 (중복 제거, 시간순 정렬)

1. **편집 → 알림 키워드 설정**
2. 키워드 입력: `긴급, 중요`
3. 해당 키워드가 포함된 자막 감지 시 토스트 알림 표시

#### 트레이 모드
1. **보기 → 트레이로 최소화** 활성화
2. 창을 닫으면 트레이로 최소화됨
3. 트레이 아이콘 더블클릭으로 복원

### 연결 상태 모니터링 (v16.6)

### 자동 재연결 (v16.6)
연결이 끊기면 자동으로 재연결을 시도합니다.

- **지수 백오프**: 2초 → 4초 → 8초 → 16초 → 32초
- **최대 시도**: 5회
- 재연결 성공/실패 시 토스트 알림 표시

### 생중계 자동 감지
xcode만 포함된 URL로 접속하면 프로그램이 자동으로 현재 진행 중인 생중계를 찾습니다.

**작동 원리:**
1. `live_list.asp` API로 진행 중인 방송 목록 조회
2. 페이지의 JavaScript 변수에서 xcgcd 확인
3. URL 리다이렉트 감지
4. 페이지 내 링크/iframe/input 요소 파싱
5. 메인 페이지에서 해당 위원회의 생중계 버튼 자동 클릭

감지에 실패하면 `📡 생중계 목록` 버튼으로 현재 방송을 직접 선택할 수 있습니다.

### 자막 파이프라인 고정 구조 (중요)
현재 자막 수집 파이프라인은 아래 구조로 **고정**되어 있습니다.

1. Worker가 0.2초 간격으로 raw 텍스트를 읽고, compact 기준으로 중복 전송을 억제
2. UI Queue에서 `preview` 메시지를 `_prepare_preview_raw`로 정규화/게이팅
3. 통과된 입력만 `_process_raw_text`(글로벌 히스토리 + suffix)로 전달
4. 후단에서 `get_word_diff`/recent compact tail 체크로 대량 반복 블록 재누적을 한 번 더 차단
5. 종료 시 `_drain_pending_previews`에서 남은 preview/segments를 소진하고 마지막 항목을 보정

운영 원칙:
- 코어 알고리즘(`_process_raw_text`, `_extract_new_part`)은 직접 수정하지 않음
- 반복/누락 이슈는 우선 게이트 임계값과 fallback 경로에서 조정
- 로그 키워드: `preview suffix desync reset`, `preview ambiguous suffix reset`

### 상임위원회 프리셋
20개 상임위원회의 URL이 내장되어 있습니다:

- 본회의, 국회운영위원회, 법제사법위원회, 정무위원회
- 재정경제기획위원회, 교육위원회, 과학기술정보방송통신위원회
- 외교통일위원회, 국방위원회, 행정안전위원회
- 문화체육관광위원회, 농림축산식품해양수산위원회
- 산업통상자원중소벤처기업위원회, 보건복지위원회

## 🔧 문제 해결

### Chrome 시작 오류
- Chrome 브라우저가 최신 버전인지 확인
- 프로그램이 자동으로 최대 3회 재시도합니다
- Selenium 4.x가 WebDriver를 자동 관리합니다

### 자막이 표시되지 않음
1. 국회 의사중계 페이지에서 자막이 활성화되어 있는지 확인
2. 다른 CSS 선택자 시도: `#viewSubtit`, `.subtitle_area`
3. 생중계가 진행 중인지 확인
4. URL에 `xcgcd`가 포함되어 있는지 확인 (또는 `📡 생중계 목록`에서 선택)

### 연결이 자주 끊김
- v16.6부터 자동 재연결이 지원됩니다
- 상태바에서 연결 상태(🟢/🔴/🟡) 확인
- 네트워크 환경 점검 필요

### HWP 저장 오류
- **한글 오피스**가 설치되어 있어야 합니다
- 저장 실패 시 RTF/DOCX/TXT 대체 형식을 선택할 수 있습니다
- RTF 파일은 한글에서 열 수 있습니다
