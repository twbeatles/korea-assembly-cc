## ✨ v16.9 새 기능

### 🛡️ 데이터 무결성 및 검증 (v16.9)
- **중복 감지 알고리즘 강화** - 단순 텍스트 비교 대신 Jaccard 유사도 분석으로 미세한 수정/중복 감지
- **백업 파일 무결성 검증** - 자동 백업 시 임시 파일 검증 단계를 거쳐 손상 방지
- **세션 로드 복구** - JSON 파싱 오류 시 백업 폴더 열기 안내

### 🔒 안정성 및 성능 (v16.9)
- **스레드 안전성 전면 강화** - 자막 저장/편집/초기화 시 `subtitle_lock` 적용으로 충돌 방지
- **DB 최적화** - 연결 캐싱(Pooling) 및 Bulk Insert 최적화, FTS5 검색 엔진 적용
- **자막 렌더링 가속** - 전체 다시 그리기 대신 증분 렌더링 도입
- **스크롤 우선 표시** - 사용자가 스크롤로 보고 있을 때 자동 스크롤을 일시 중지하고 위치 유지
- **분 단위 자막 묶기** - 같은 분은 합치고, 길이/간격이 크면 분리하여 안정적으로 저장
- **중지 즉시 확정** - 중지 시 진행 중인 자막을 즉시 확정하고 큐를 정리해 종료 안정성 개선

### 🎨 UI/UX 개선 (v16.9)
- **키워드 입력 최적화** - 디바운싱(Debouncing) 적용으로 입력 중 렉 제거
- **병합 옵션 세분화** - 병합 시 현재 세션 자막 보존 여부 선택 가능
- **HWP 저장 개선** - 저장 실패 시 RTF/DOCX 등 대체 포맷 제안

### 📡 생중계 목록 선택 (v16.7)
- **생중계 목록 UI** - `📡 생중계 목록` 버튼으로 현재 방송 목록 확인
- **상태/시작 시간 표시** - 진행중/예정/종료 상태와 시작 시간 노출
- **선택 즉시 접속** - 선택한 방송의 `xcode+xcgcd` URL 자동 구성

### 🧭 URL 감지 강화 (v16.7)
- `live_list.asp` API와 페이지 분석으로 **xcgcd 자동 보완**
- 메인 페이지 리다이렉트 시 **해당 위원회 생중계 버튼 자동 클릭**
- 실패 시 안내 메시지 개선 + 생중계 목록 버튼 안내

### 🧾 상임위원회 xcode 최신화 (v16.8)
- **재정경제기획위원회(65), 성평등가족위원회(63), 정무위원회(26)** 반영
- **명칭 변경/약칭 대응** (기재위/재경위, 환노위, 여가위 등)

### ⚡ 성능 최적화 (v16.8)
- **정규식/포맷 캐싱**으로 렌더링/정제 속도 개선
- **통계 캐시**로 긴 세션에서도 UI 응답성 유지

### 🔌 연결 상태 모니터링 (v16.6)
- **실시간 연결 상태 표시** - 상태바에 🟢/🔴/🟡 아이콘으로 표시
- **응답 시간 표시** - 마우스를 올리면 현재 latency 확인 가능

### 🔄 자동 재연결 (v16.6)

- **최대 5회 재시도** - 실패 시 알림 표시
- 재연결 성공 시 토스트 알림

### 📝 자동 파일명 생성 (v16.6)
- **스마트 파일명** - `20260122_법제사법위원회_134500.txt` 형식
- URL에서 위원회명 자동 추출
- TXT/SRT/VTT/DOCX/RTF 저장에 적용 (HWP는 기본 파일명 사용)

### 🗄️ SQLite 데이터베이스 (v16.6)
- **세션 히스토리** - 모든 추출 세션 자동 저장
- **통합 검색** - 전체 자막에서 키워드 검색
- **통계 조회** - 총 세션 수, 자막 수, 글자 수 확인
- 메뉴: `데이터베이스` → 세션 히스토리 / 자막 검색 / 전체 통계

### 📎 자막 병합 (v16.6)
- **다중 세션 병합** - 여러 세션 파일을 하나로 합치기
- **중복 제거** - 동일 자막 자동 필터링
- **시간순 정렬** - 옵션으로 정렬 여부 선택
- 메뉴: `도구` → 자막 병합 (Ctrl+Shift+M)

### 2. 필수 라이브러리 설치
```bash
pip install PyQt6 selenium
```

## 🚀 사용 방법

### 기본 사용법

#### 1단계: 프로그램 실행
```bash
python "국회의사중계 자막.py"
```

#### 2단계: 위원회 선택
세 가지 방법 중 선택:

**방법 A: 프리셋 사용 (권장)**
1. `📋 상임위` 버튼 클릭
2. 원하는 위원회 선택 (예: 본회의, 법제사법위원회)
3. URL이 자동으로 입력됨

**방법 B: 직접 URL 입력**
1. URL 입력창에 국회 의사중계 URL 입력
2. 예: `https://assembly.webcast.go.kr/main/player.asp?xcode=10`

**방법 C: 생중계 목록 선택 (v16.7)**
1. `📡 생중계 목록` 버튼 클릭
2. 목록에서 현재 방송 선택
3. URL이 자동으로 입력됨

#### 3단계: 옵션 설정
- ✅ **자동 스크롤**: 새 자막이 추가될 때 자동으로 아래로 스크롤 (사용자 스크롤 시 일시 중지)
- ✅ **실시간 저장**: 자막을 파일에 실시간으로 저장 (realtime_output 폴더)
- ✅ **헤드리스 모드**: 브라우저 창을 숨기고 백그라운드에서 실행

#### 4단계: 추출 시작
- `▶ 시작` 버튼 클릭 또는 **F5** 키
- 상태바에서 진행 상황 확인:
  - `🔍 현재 생중계 감지 중...` - xcgcd 자동 탐지 진행
  - `✅ 생중계 감지 성공!` - 오늘 생중계 발견
  - `자막 모니터링 중` - 정상 동작 중
- 연결 상태 표시: 🟢 연결됨, 🔴 끊김, 🟡 재연결 중

#### 5단계: 자막 확인
- 왼쪽 패널에 자막이 실시간으로 표시됨
- 오른쪽 통계 패널에서 글자수, 단어수, 실행 시간 확인
- `⏳` 표시는 아직 확정되지 않은 진행 중 자막

#### 6단계: 저장
추출 완료 후 다양한 형식으로 저장 (자동 파일명 제안됨):
- **파일 → TXT 저장** (Ctrl+S) - 일반 텍스트
- **파일 → SRT 저장** - 자막 파일 형식
- **파일 → DOCX 저장** - Word 문서
- **파일 → 세션 저장** (Ctrl+Shift+S) - 나중에 다시 불러오기 가능 + DB 저장

### 고급 사용법

#### 데이터베이스 활용 (v16.6 신규)
1. **세션 히스토리**: `데이터베이스` → `📋 세션 히스토리`
   - 저장된 모든 세션 목록 확인
   - 세션 선택 후 불러오기/삭제
2. **자막 검색**: `데이터베이스` → `🔍 자막 검색`
   - 전체 세션에서 키워드 통합 검색
3. **통계 확인**: `데이터베이스` → `📊 전체 통계`

#### 자막 병합 (v16.6 신규)
1. **도구 → 📎 자막 병합...** (Ctrl+Shift+M)
2. 병합할 세션 파일들 추가
3. 옵션 선택 (중복 제거, 시간순 정렬)
4. "병합 실행" 클릭

#### 키워드 하이라이트
1. **편집 → 하이라이트 키워드 설정**
2. 키워드 입력 (쉼표로 구분): `법안, 통과, 의결`
3. 해당 키워드가 노란색으로 강조됨

#### 알림 키워드
1. **편집 → 알림 키워드 설정**
2. 키워드 입력: `긴급, 중요`
3. 해당 키워드가 포함된 자막 감지 시 토스트 알림 표시

#### 트레이 모드
1. **보기 → 트레이로 최소화** 활성화
2. 창을 닫으면 트레이로 최소화됨
3. 트레이 아이콘 더블클릭으로 복원

## 📚 상세 기능 설명

### 연결 상태 모니터링 (v16.6)
상태바에서 WebDriver 연결 상태를 실시간으로 확인할 수 있습니다.

| 아이콘 | 상태 | 설명 |
|--------|------|------|
| 🟢 | 연결됨 | 정상 작동 중, 툴팁에 응답 시간 표시 |
| 🔴 | 끊김 | 네트워크 오류 감지 |
| 🟡 | 재연결 중 | 자동 재연결 시도 중 |

### 자동 재연결 (v16.6)
연결이 끊기면 자동으로 재연결을 시도합니다.

- **지수 백오프**: 2초 → 4초 → 8초 → 16초 → 32초
- **최대 시도**: 5회
- 재연결 성공/실패 시 토스트 알림 표시

### 생중계 자동 감지
xcode만 포함된 URL로 접속하면 프로그램이 자동으로 현재 진행 중인 생중계를 찾습니다.

**작동 원리:**
1. `live_list.asp` API로 진행 중인 방송 목록 조회
2. 페이지의 JavaScript 변수에서 xcgcd 확인
3. URL 리다이렉트 감지
4. 페이지 내 링크/iframe/input 요소 파싱
5. 메인 페이지에서 해당 위원회의 생중계 버튼 자동 클릭

감지에 실패하면 `📡 생중계 목록` 버튼으로 현재 방송을 직접 선택할 수 있습니다.

### 상임위원회 프리셋
20개 상임위원회의 URL이 내장되어 있습니다:

- 본회의, 국회운영위원회, 법제사법위원회, 정무위원회
- 재정경제기획위원회, 교육위원회, 과학기술정보방송통신위원회
- 외교통일위원회, 국방위원회, 행정안전위원회
- 문화체육관광위원회, 농림축산식품해양수산위원회

## 🔧 문제 해결

### Chrome 시작 오류
- Chrome 브라우저가 최신 버전인지 확인
- 프로그램이 자동으로 최대 3회 재시도합니다
- Selenium 4.x가 WebDriver를 자동 관리합니다

### 자막이 표시되지 않음
1. 국회 의사중계 페이지에서 자막이 활성화되어 있는지 확인
2. 다른 CSS 선택자 시도: `#viewSubtit`, `.subtitle_area`
3. 생중계가 진행 중인지 확인
4. URL에 `xcgcd`가 포함되어 있는지 확인 (또는 `📡 생중계 목록`에서 선택)

### 연결이 자주 끊김
- v16.6부터 자동 재연결이 지원됩니다
- 상태바에서 연결 상태(🟢/🔴/🟡) 확인
- 네트워크 환경 점검 필요

### HWP 저장 오류
- **한글 오피스**가 설치되어 있어야 합니다
- COM 오류 시 자동으로 RTF 형식으로 저장됩니다
- RTF 파일은 한글에서 열 수 있습니다
